# commonMistakes

Purpose: AI/Codex가 반복적으로 내는 실수를 사전에 차단한다.

## 1. 로직/설계 실수
- 서비스 책임을 컨트롤러로 올려버리는 실수
- 일반가입/초대가입 같은 다른 유즈케이스를 한 메서드로 뭉개는 실수
- `shareUuid` 검증 전에 사용자 생성을 먼저 해버리는 실수

## 2. DB/엔티티 실수
- `ddl-auto=validate` 환경에서 엔티티 컬럼명 누락/불일치
- 감사 컬럼(non-null) 고려 없이 데이터 저장 시도
- soft delete(`deleted_at`) 상태 검증 누락

## 3. API 실수
- 성공 응답 코드를 임의로 `200`으로 통일
- 에러 응답 포맷이 엔드포인트마다 달라지는 문제
- 정책 변경(예: invite-register only) 후 레거시 엔드포인트를 남겨두는 실수

## 4. 테스트 실수
- 신규 기능은 테스트했지만 기존 흐름 회귀를 확인하지 않는 실수
- integration test에서 실제 DB 전제(`replace = NONE`)를 놓치는 실수
- 실패 케이스(만료 링크, 사용횟수 초과, 비밀번호 불일치) 누락

## 5. 문서 실수
- 코드 변경 후 handoff/brief 문서를 갱신하지 않는 실수
- 구현 사실이 아닌 계획/추측을 완료된 것처럼 기록하는 실수
